# Panel Client

Клиент-плеер для цифровых экранов. Устройство один раз настраивается, отправляет заявку на регистрацию, после подтверждения сервером переходит в рабочий режим.

## Требования

- Flutter stable
- Windows / Linux / macOS
- Доступ к серверу по `http://IP:443` через frontend-proxy
- Или по `http://IP:8000`, если backend опубликован напрямую

## Быстрый запуск

### Windows

```bash
flutter pub get
flutter run -d windows
```

### Linux

```bash
flutter pub get
flutter run -d linux
```

### Сборка Windows

```bash
flutter build windows
```

Готовый `.exe` будет в:

```text
build/windows/x64/runner/Release/
```

## Первый запуск устройства

При первом запуске клиент открывает мастер настройки.

Нужно указать:

- адрес сервера, например `http://192.168.1.50:443`
- если backend открыт напрямую без frontend-proxy, можно использовать `http://192.168.1.50:8000`
- имя устройства

Дальше порядок такой:

1. Нажать `Проверить соединение`
2. Нажать `Отправить заявку`
3. Подтвердить устройство на сайте в списке ожидающих
4. После подтверждения клиент автоматически перейдёт в рабочий режим

Важно:

- вводить нужно только адрес сервера
- не нужно указывать `/api/...`
- клиент сам добавит нужные API-пути
- если указан только IP/домен без порта, клиент сначала пробует `http://host`, а затем автоматически пробует `http://host:443`

## Что создаёт клиент локально

```text
Documents/panel/
├── config.json
├── device.json
├── manifest.json
├── log.txt
└── media/
```

## Что хранится в config.json

Пример:

```json
{
  "media_root": "C:\\Users\\<user>\\Documents\\panel\\media",
  "server_url": "http://192.168.1.50:8000",
  "api_base": "http://192.168.1.50:8000/api/v1/device"
}
```

`api_base` формируется автоматически. Основной параметр для пользователя это `server_url`.

## Как работает клиент

- отправляет заявку на регистрацию:
  - `POST /api/v1/device/register/request`
- проверяет статус регистрации:
  - `GET /api/v1/device/register/status`
- после подтверждения получает манифест:
  - `GET /api/v1/device/manifest`
- отправляет heartbeat:
  - `POST /api/v1/device/heartbeat`
- скачивает контент через backend:
  - `GET /api/v1/device/media/{id}`

Клиент не должен ходить напрямую в MinIO/S3.
